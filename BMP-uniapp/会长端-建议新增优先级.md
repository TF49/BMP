# 会长端建议新增优先级

本文档用于整理当前 uniapp 会长端仍需新增或补齐的关键页面，重点按业务优先级排序，便于后续排期开发。

## 一、优先级总览

| 优先级 | 页面 | 当前状态 | 建议原因 | 建议处理方式 |
| --- | --- | --- | --- | --- |
| P0 | `pages/president/course-booking/detail` | 缺失 | 课程预约管理如果只有列表，没有详情页，就无法承接核销、取消、查看报名人、查看预约状态等关键操作 | 新增真实详情页，并支持状态处理 |
| P0 | `pages/president/equipment-rental/detail` | 缺失 | 器材租借是典型订单流，必须查看借出人、借出时间、归还状态、逾期情况 | 新增真实详情页，并支持归还/逾期处理 |
| P0 | `pages/president/stringing/detail` | 缺失 | 穿线服务本质上是工单管理，没有详情页就无法推进状态、修改价格、记录备注 | 新增真实工单详情页 |
| P0 | `pages/president/tournament-registration/detail` | 缺失 | 赛事报名需要审核参赛人信息、确认报名状态、处理名单导出前校验 | 新增真实报名详情/审核页 |
| P1 | `pages/president/notification/detail` | 缺失 | 通知发布后需要查看内容、发布时间、对象范围，后续也可能要支持撤回或二次编辑 | 新增通知详情页 |
| P1 | `pages/president/finance/audit-log` | 缺失 | 财务后台通常需要操作留痕与审计查看，移动端可先做精简版 | 新增财务审计日志页 |
| P1 | `pages/president/court/detail` | 缺失 | 场地信息如果后续包含价格、状态、维护记录、预约概览，详情页比直接在列表里处理更清晰 | 新增场地详情页 |
| P2 | `pages/president/equipment-rental/form` | 缺失 | 如果业务需要后台人工创建租借单，则需要独立表单；如果不需要，可暂缓 | 视业务决定是否新增 |
| P2 | `pages/president/stringing/form` | 缺失 | 如果会长端需要手动录入穿线服务单，则需要表单页；否则列表+详情即可 | 视业务决定是否新增 |
| P2 | 独立 `notification/edit` 页面 | 缺失 | 如果现有 `notification/form` 可以兼容新增和编辑，则无需单独做编辑页 | 优先复用 `form`，一般不单独新增 |

## 二、建议优先补齐的占位模块

以下页面虽然已经存在，但当前仍是占位页，建议优先转为真实业务页面：

| 页面 | 当前状态 | 建议 |
| --- | --- | --- |
| `pages/president/course-booking/list` | 占位 | 尽快改为真实预约列表，并配套 `detail` 页 |
| `pages/president/equipment-rental/list` | 占位 | 尽快改为真实租借列表，并配套 `detail` 页 |
| `pages/president/stringing/list` | 占位 | 尽快改为真实工单列表，并配套 `detail` 页 |
| `pages/president/tournament-registration/list` | 占位 | 尽快改为真实报名审核列表，并配套 `detail` 页 |
| `pages/president/notification/list` | 占位 | 尽快改为真实通知列表，并配套 `detail` 页 |
| `pages/president/notification/form` | 占位 | 优先改为真实发布/编辑页 |
| `pages/president/court/form` | 占位 | 直接补成真实新增/编辑页 |

## 三、推荐开发顺序

建议按下面顺序推进：

1. 先补齐订单/工单型详情页：`course-booking/detail`、`equipment-rental/detail`、`stringing/detail`、`tournament-registration/detail`
2. 再把对应占位列表页改成真实列表页：`course-booking/list`、`equipment-rental/list`、`stringing/list`、`tournament-registration/list`
3. 然后补通知模块：`notification/list`、`notification/form`、`notification/detail`
4. 最后补后台增强页：`finance/audit-log`、`court/detail`

## 四、补充说明

### 1. 更适合“补功能”而不是“新增页面”的模块

| 模块 | 当前情况 | 建议 |
| --- | --- | --- |
| 会员管理 | `list/detail/form/recharge` 已存在 | 优先补详情页内部功能，如约球记录、余额调整 |
| 场地管理 | `court/form` 已存在但为占位 | 优先把现有表单页做实，不必另起新页 |
| 通知管理 | `notification/form` 已存在但为占位 | 优先复用并做成真实新增/编辑页 |
| 器材库存 | `equipment/stock` 已存在 | 优先接真实库存流水和日志数据 |

### 2. 页面新增原则

- 列表页适合承接筛选、搜索、批量查看
- 详情页适合承接审核、状态变更、查看完整信息
- 表单页适合承接新增与编辑
- 如果一个模块主要是“查看 + 状态处理”，通常优先做“列表 + 详情”
- 如果一个模块主要是“录入 + 修改”，通常优先做“列表 + 表单”

## 五、结论

当前会长端最需要补齐的不是更多入口页，而是把已有占位模块补成真实业务闭环，尤其是：

- 课程预约详情
- 器材租借详情
- 穿线工单详情
- 赛事报名详情
- 通知详情

其中前四项建议作为第一优先级处理。
