# 会长端建议新增优先级

本文档用于整理当前 uniapp 会长端仍需新增或补齐的关键页面，重点按业务优先级排序，便于后续排期开发。

**最近同步（2026-04）：** 下列 P0 详情页已落地 **初版 Vue 页面**（`pages.json` / `src/pages.json` 已注册，`utils/presidentRouter.ts` 已增加对应常量；占位列表页提供「查看详情（示例）」入口）。当前为 **Mock 数据 + 交互占位**，**后端接口、真实列表与权限**仍需对接。

## 一、优先级总览

| 优先级 | 页面 | 当前状态 | 建议原因 | 建议处理方式 |
| --- | --- | --- | --- | --- |
| P0 | `pages/president/course-booking/detail` | **初版已有**（Mock，待接 API） | 课程预约管理如果只有列表，没有详情页，就无法承接核销、取消、查看报名人、查看预约状态等关键操作 | 对接预约详情/核销/取消接口；列表改为真实数据并 `navigateTo` 传 `id` |
| P0 | `pages/president/equipment-rental/detail` | **初版已有**（Mock，待接 API） | 器材租借是典型订单流，必须查看借出人、借出时间、归还状态、逾期情况 | 对接租借详情与归还/逾期接口；列表联调 |
| P0 | `pages/president/stringing/detail` | **初版已有**（Mock，待接 API） | 穿线服务本质上是工单管理，没有详情页就无法推进状态、修改价格、记录备注 | 对接工单详情与进度更新/打印（或导出）接口；列表联调 |
| P0 | `pages/president/tournament-registration/detail` | **初版已有**（Mock，待接 API） | 赛事报名需要审核参赛人信息、确认报名状态、处理名单导出前校验 | 对接报名详情与审核接口；列表联调 |
| P1 | `pages/president/notification/detail` | 缺失 | 通知发布后需要查看内容、发布时间、对象范围，后续也可能要支持撤回或二次编辑 | 新增通知详情页 |
| P1 | `pages/president/finance/audit-log` | 缺失 | 财务后台通常需要操作留痕与审计查看，移动端可先做精简版 | 新增财务审计日志页 |
| P1 | `pages/president/court/detail` | 缺失 | 场地信息如果后续包含价格、状态、维护记录、预约概览，详情页比直接在列表里处理更清晰 | 新增场地详情页 |
| P2 | `pages/president/equipment-rental/form` | 缺失 | 如果业务需要后台人工创建租借单，则需要独立表单；如果不需要，可暂缓 | 视业务决定是否新增 |
| P2 | `pages/president/stringing/form` | 缺失 | 如果会长端需要手动录入穿线服务单，则需要表单页；否则列表+详情即可 | 视业务决定是否新增 |
| P2 | 独立 `notification/edit` 页面 | 缺失 | 如果现有 `notification/form` 可以兼容新增和编辑，则无需单独做编辑页 | 优先复用 `form`，一般不单独新增 |

建议按下面顺序推进：

1. P0 详情初版已完成；优先为上述四模块 **对接后端**，并将占位列表改为真实列表（`navigateTo` 传 `id`）。

## 四、补充说明

### 1. 更适合“补功能”而不是“新增页面”的模块

| 模块 | 当前情况 | 建议 |
| --- | --- | --- |
| 会员管理 | `list/detail/form/recharge` 已存在 | 优先补详情页内部功能，如约球记录、余额调整 |
| 场地管理 | `court/form` 已存在但为占位 | 优先把现有表单页做实，不必另起新页 |
| 通知管理 | `notification/form` 已存在但为占位 | 优先复用并做成真实新增/编辑页 |
| 器材库存 | `equipment/stock` 已存在 | 优先接真实库存流水和日志数据 |

### 2. 页面新增原则

- 列表页适合承接筛选、搜索、批量查看
- 详情页适合承接审核、状态变更、查看完整信息
- 表单页适合承接新增与编辑
- 如果一个模块主要是“查看 + 状态处理”，通常优先做“列表 + 详情”
- 如果一个模块主要是“录入 + 修改”，通常优先做“列表 + 表单”

### 3. 路由常量（会长端）

以下路径已在 `utils/presidentRouter.ts` 中配置，列表与工作台跳转时请优先使用常量，避免硬编码：

- `COURSE_BOOKING_DETAIL`
- `EQUIPMENT_RENTAL_DETAIL`
- `STRINGING_DETAIL`
- `TOURNAMENT_REGISTRATION_DETAIL`

## 五、结论

当前会长端 **P0 四类订单/工单/报名详情** 已具备 **可演示的初版界面与路由闭环（Mock）**，仍需 **接口与列表** 打通才能形成真实业务闭环。后续优先项：

- 四个占位 **列表页** 接真实数据并跳转详情
- **通知详情**（P1）及通知列表/表单做实
- 财务审计、场地详情等 **P1** 增强页按排期补充

其中 **通知模块** 与 **列表+接口联调** 建议作为下一批重点。
