## 1. 系统概览

### 1.1 项目定义

羽擎（Badminton Management Platform，BMP）是一个面向羽毛球场馆经营场景的单仓多端业务系统，当前同时覆盖：

- 场馆、场地、会员、教练、课程、赛事、器材等基础经营资源
- 预约、租借、报名、穿线、充值、消费、财务等交易与服务流程
- Web 官网、Web 管理端、Web 用户端、Web 教练端、UniApp 移动多角色入口

## 5. 多端结构

### 5.1 Web 端

当前 `vue/src/router/index.js` 的真实结构可以归纳为四层：

| 场景  | 入口 / 路由组                                                      | 说明                             |
| --- | ------------------------------------------------------------- | ------------------------------ |
| 官网  | `/site`                                                       | 公共品牌页，根路径强制先跳这里                |
| 管理端 | `/dashboard`、`/venue/*`、`/court/*`、`/member/*`、`/finance/*` 等 | 面向 `PRESIDENT`、`VENUE_MANAGER` |
| 用户端 | `/user/*`                                                     | 用户自助业务流                        |
| 教练端 | `/coach/*`                                                    | 教练工作台、我的课程、课表、预约、档案            |

当前路由守卫还做了三件事：

1. `ADMIN` 角色在前端统一映射为 `PRESIDENT`
2. `USER` / `MEMBER` 访问管理路由时自动回跳用户端
3. `COACH` 访问用户端或管理端时自动回跳教练端

### 5.2 UniApp 端

当前 `BMP-uniapp/pages.json` 与目录结构可以归纳为三层：

| 页面组     | 目录                                                                                                                                 | 说明                               |

| 用户端与公共页 | `pages/index`、`pages/venue`、`pages/booking`、`pages/course`、`pages/equipment`、`pages/tournament`、`pages/profile`、`pages/settings` 等 | 用户主链路                            |

## 6. 权限体系

### 6.1 角色定义

| 角色    | 代码              | 说明        |
| ----- | --------------- | --------- |
| 协会会长  | `PRESIDENT`     | 系统最高权限    |
| 场馆管理者 | `VENUE_MANAGER` | 仅管理所属场馆数据 |
| 普通用户  | `USER`          | 用户端基础身份   |
| 会员    | `MEMBER`        | 用户端会员身份   |
| 教练    | `COACH`         | 教练本人工作流身份 |

### 6.2 权限实现

系统权限控制分为三层：

1. **Controller 层**：使用 `@PreAuthorize` 做入口鉴权
2. **Service 层**：按当前登录人补充场馆或本人数据范围限制
3. **前端层**：通过路由守卫与菜单权限控制可见性

### 6.3 权限矩阵摘要

| 功能模块              |  会长 | 场馆管理者 | 用户/会员 |    教练    |
| ----------------- | :-: | :---: | :---: | :------: |
| 用户管理              |  ✅  |   ❌   |   ❌   |     ❌    |
| 场馆管理              |  ✅  |   ❌   |  👁️  |    👁️   |
| 场地管理              |  ✅  |   🏢  |  👁️  |    👁️   |
| 会员管理              |  ✅  |   🏢  |   👤  | 🏢 范围内查看 |
| 财务管理              |  ✅  |   🏢  |   ❌   |     ❌    |
| 预约 / 报名 / 租借 / 穿线 |  ✅  |   🏢  |   👤  | 课程相关本人处理 |
| 教练档案              |  ✅  |   🏢  |  👁️  |    👤    |

图例：

- `✅`：完整管理
- `🏢`：仅所属场馆
- `👤`：仅本人数据
- `👁️`：只读或辅助查看
- `❌`：无权限

## 8. 当前边界与注意事项

### 8.1 后端

- 当前没有 `src/test`，稳定性仍偏依赖编译验证与前端联调
- 活动流由 `member`、`booking`、`finance` 三类数据聚合生成，没有独立活动表

### 8.2 Web

- 当前一个工程同时承载官网、后台、用户端、教练端，构建产物已经偏大
- `npm run build` 可通过，但 `Element Plus`、`ECharts`、`vendors` 分包都需要后续治理

### 8.3 UniApp

- 当前主干 `npm run type-check` 已通过，不再是阻塞项
- `build:mp-weixin` 通过，但仍有 Sass `legacy-js-api` 和 `@import` 弃用告警
- 真实设备和多角色链路仍建议继续做手测回归
