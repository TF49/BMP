## 1. 系统概览

### 1.1 项目定义

羽擎（Badminton Management Platform，简称 BMP）是一个面向羽毛球场馆经营场景的单仓多端业务系统，覆盖以下核心能力：

- 场馆、场地、会员、教练、课程、赛事、器材等基础业务管理
- 预约、租借、报名、充值、消费、财务等交易类业务流转
- Web 管理端、Web 用户端、Web 教练端、UniApp 移动端的多角色协同

### 1.2 系统组成

BMP 当前由三部分组成：

1. **后端**：`Spring Boot` 单体应用，承载统一业务核心与数据权限控制  
2. **Web 端**：`Vue 3` 工程，承载官网、管理端、用户端、教练端  
3. **UniApp 端**：移动端工程，承载用户端、移动管理端、教练端入口

### 1.3 当前实现范围

从当前仓库可以确认：

- 后端核心业务链路已成型
- Web 多角色场景已成型
- UniApp 多角色入口已落地
- 系统已具备认证、权限、推送、状态流转、财务联动等持续运行型能力

## 3. 启动与运行

### 3.1 后端

```bash
mvn spring-boot:run
```

- 默认端口：`9090`
- API 前缀：`/api`
- Swagger UI：`/swagger-ui.html`

### 3.2 Web 端

```bash
cd vue
npm install
npm run serve
```

- 默认端口：`8080`
- 开发环境通过 `vue.config.js` 代理 `/api` 与 `/ws`

### 3.3 UniApp 端

```bash
cd BMP-uniapp
npm install
npm run dev:mp-weixin
```

## 6. 权限体系

### 6.1 角色定义

| 角色 | 代码 | 说明 |
|------|------|------|
| 协会会长 | `PRESIDENT` | 系统最高权限 |
| 场馆管理者 | `VENUE_MANAGER` | 仅管理所属场馆数据 |
| 普通用户 | `USER` | 用户端基础身份 |
| 会员 | `MEMBER` | 用户端会员身份，与 USER 同角色侧 |
| 教练 | `COACH` | 教练本人工作流身份 |

### 6.2 权限实现方式

系统权限控制分为三层：

1. **Controller 层**：使用 `@PreAuthorize` 进行入口鉴权  
2. **Service 层**：基于当前登录人补充数据范围校验  
3. **前端层**：通过路由守卫与按钮级权限指令限制交互入口

### 6.3 权限矩阵摘要

| 功能模块 | 会长 | 场馆管理者 | 用户/会员 | 教练 |
|----------|:----:|:----------:|:---------:|:----:|
| 用户管理 | ✅ | ❌ | ❌ | ❌ |
| 场馆管理 | ✅ | ❌ | 👁️ | 👁️ |
| 会员管理 | ✅ | 👁️ | 👤 | 👁️ |
| 场地管理 | ✅ | 🏢 | 👁️ | 👁️ |
| 预约管理 | ✅ | 🏢 | 👤 | 👁️ 本人课程预约 |
| 教练管理 | ✅ | 🏢 | 👁️ | 👤 本人档案 |
| 课程管理 | ✅ | 🏢 | 👁️ | 👁️ / 本人课程视图 |
| 财务管理 | ✅ | 🏢 | ❌ | ❌ |

图例说明：

- `✅`：完全管理
- `🏢`：仅限所属场馆
- `👤`：仅限本人数据
- `👁️`：只读或查看
- `❌`：无权限

## 10. 近期重点变更

### 10.1 场地预约模型重构

当前预约模块已经从“单场地独占预约”升级为“预约主单 + 场地明细”的结构：

- 主单：`biz_booking`
- 明细：`biz_booking_court`

后端当前只支持两种预约模式：

- `SHARED`：拼场
- `PACKAGE`：包场

当前真实业务规则如下：

1. `SHARED` 仅允许选择 1 块场地
2. `PACKAGE` 允许选择多块场地，但不能包下整个场馆全部场地
3. 同场地同时间段下：
   - `SHARED` 与 `SHARED` 可叠加
   - `PACKAGE` 与任意模式互斥
4. 若某场地该时段已有任意有效预约，则不能加入新的 `PACKAGE`
5. 若某场地该时段已存在 `PACKAGE`，则不能再创建新的 `SHARED`
6. `PACKAGE` 需至少提前 2 小时提交

### 10.2 场地价格模型升级

当前场地价格配置已经拆分为三套：

- `packagePricePerHour`：包场每小时
- `sharedPricePerHour`：拼场每小时
- `sharedPricePerTime`：拼场按次

前端与后端当前统一支持三种计费方式：

- `PACKAGE_HOUR`
- `SHARED_HOUR`
- `SHARED_TIME`

### 10.3 多端预约页改造现状

当前两套前端均已接入新预约模型：

- Web：
  - 用户预约页
  - 管理端预约管理页
  - 场地价格配置页
- UniApp：
  - 用户预约页
  - 移动管理端新建预约页
  - 预约列表 / 预约详情 / 订单确认页
  - 移动端场地价格配置页

本轮 UniApp 改造遵循“保留原页面骨架、只做增量补功能”的约束，没有再整体重写预约页面样式。
