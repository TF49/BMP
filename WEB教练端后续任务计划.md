# Web 教练端后续任务计划

## 一、当前已完成

### P0 预约闭环
- 教练可查看自己课程下的预约列表
- 支持预约详情查看
- 支持签到上课：`已支付 -> 进行中`
- 支持完成课程：`进行中 -> 已完成`
- 支持缺席/取消：`已支付/进行中 -> 已取消`
- 教练权限已与管理员、普通用户隔离
- 已补充支付状态展示
- 已补充“已取消但已支付需管理员退款”的提示
- 已补充详情失败态、空态与状态更新反馈

### 当前验证情况
- 后端 `mvn -q -DskipTests compile` 通过
- 前端 `npm.cmd run build` 通过
- 自动化测试暂未纳入本轮

## 二、P1 任务

### 1. 我的课程增强
目标：让“我的课程”从查询页升级为工作页。

计划内容：
- 增加课程详情抽屉或弹窗
- 展示课程完整信息：
  - 课程名称
  - 日期
  - 时间
  - 场地
  - 当前人数 / 最大人数
  - 状态
  - 备注
- 增加“查看该课程预约”入口
- 从课程页跳转到预约页时自动带课程筛选条件
- 优化状态筛选与时间筛选交互
- 补充空态、加载态、异常提示

验收标准：
- 教练可在课程页直接查看课程完整信息
- 可从课程页一键进入对应预约明细
- 页面在无数据、加载失败时有明确反馈

### 2. 我的课表增强
目标：让“我的课表”更接近日程视角，而不是单纯表格。

计划内容：
- 增加按天 / 按周查看切换
- 支持点击课表项查看课程详情
- 支持从课表项跳转到对应预约列表
- 突出展示：
  - 今天课程
  - 进行中课程
  - 即将开始课程
- 优化日期切换与空课表提示

验收标准：
- 教练可快速查看当天和本周课程安排
- 教练可从课表直接进入课程详情或预约页
- 当日程为空时有明确空态提示

### 3. 工作台增强
目标：让首页真正成为教练工作台。

计划内容：
- 增加待处理预约数
- 增加今日进行中课程数
- 增加今日已完成课节数
- 增加最近课程卡片
- 增加最近预约卡片
- 增加快捷入口：
  - 待签到预约
  - 今日日程
- 优化未绑定教练档案时的空状态提示

验收标准：
- 工作台能直接反映当天工作重点
- 教练可从首页快速进入关键任务页面
- 未绑定场景下有明确说明和引导

### 4. 档案页增强
目标：把档案编辑体验做完整。

计划内容：
- 支持头像上传，不再只依赖手填 URL
- 增加手机号校验
- 增加专长、经验输入规则
- 增加保存中、保存成功、保存失败反馈
- 优化只读信息与可编辑信息的视觉区分

验收标准：
- 教练可直接上传头像并保存
- 表单输入错误时有明确校验提示
- 保存反馈及时明确

## 三、P2 任务

### 5. 学员相关能力
目标：补齐教练最常用的学员视角能力。

计划内容：
- 在课程详情或预约详情中查看学员名单
- 展示学员基础信息
- 支持填写学员出勤备注或课程备注
- 视情况增加历史上课记录只读查看

验收标准：
- 教练可快速查看某节课涉及的学员
- 能记录必要的课堂备注或出勤说明

### 6. 业务规则补强
目标：把边界规则和状态链路梳理扎实。

计划内容：
- 明确“已取消但已支付”状态的管理员后续处理链路
- 梳理教练端操作与自动状态更新之间的关系
- 统一预约状态文案、按钮显隐与错误提示
- 评估是否补教练端专属统计接口，减少前端拼装逻辑

验收标准：
- 不同角色对同一预约状态的理解一致
- 自动流转与手动操作不冲突
- 前后端状态文案和行为保持一致

## 四、测试与回归

目标：降低后续改动风险。

计划内容：
- 后端补教练权限测试
- 后端补预约状态流测试
- 前端补关键交互测试，或至少整理手工回归清单
- 回归检查：
  - 教练角色
  - 管理员角色
  - 普通用户角色
- 持续保留以下最低回归动作：
  - `mvn -q -DskipTests compile`
  - `npm.cmd run build`

验收标准：
- 核心权限不串角色
- 核心状态流转不回退
- 常见页面交互在回归时可快速检查

## 五、建议执行顺序

1. 我的课程增强
2. 我的课表增强
3. 工作台增强
4. 档案页增强
5. 学员相关能力
6. 业务规则补强
7. 自动化测试与回归完善

## 六、备注

- 当前 P0 功能闭环已经完成
- 本文档用于后续迭代排期与开发拆分
- 若进入下一轮实施，建议优先处理 P1，先把“课程、课表、工作台”三个高频页面做成完整工作流
