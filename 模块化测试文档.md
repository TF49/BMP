## 2. 当前自动化结果解读

### 2.1 已经确认的事情

- 后端代码当前没有编译级错误
- Web 端当前没有阻断构建的问题
- UniApp 主干当前没有阻断类型检查的问题
- UniApp 已存在基础单测，并且当前全部通过
- 小程序产物当前可以正常完成构建

### 2.2 还没有被自动化真正覆盖的事情

- 后端业务规则回归
- Web 真实页面交互回归
- UniApp 真机或开发者工具中的角色化主链路
- 多端同业务数据一致性
- 支付 / 退款 / 对账等跨模块联动的事务级回归

## 3. 当前明确的测试缺口

### 3.1 后端缺口最大

当前仓库没有 `src/test` 目录，这意味着：

- 预约、充值、财务、穿线等高耦合业务还缺少后端回归护栏
- 一旦业务规则变更，回归更多依赖人工联调
- 编译通过不等于业务正确

### 3.2 Web 构建有告警但不阻断

`vue/` 下 `npm run build` 虽然通过，但当前构建仍提示：

- `app` 入口约 `2.27 MiB`
- `chunk-element-plus` 约 `1.0 MiB`
- `chunk-echarts` 约 `896 KiB`
- `chunk-vendors` 约 `776 KiB`

这类问题不属于“测试失败”，但属于需要持续关注的交付质量问题。

### 3.3 UniApp 构建有 Sass 技术债

`BMP-uniapp/` 下 `npm run build:mp-weixin` 通过，但仍有：

- Sass `legacy-js-api` 弃用告警
- `uview-plus/index.scss` 的 `@import` 弃用告警
- `App.vue` 内部样式 `@import` 弃用告警

这类问题目前不阻塞交付，但不建议长期忽略。

## 4. 推荐回归清单

建议之后每次较大的业务改动，都至少按角色走一遍下面这些链路。

### 4.1 会长 / 场馆管理者

- 登录、退出、刷新 Token
- Dashboard 汇总
- 场馆管理、场地管理、会员管理
- 财务列表、趋势、业务占比
- 财务审计日志导出
- 财务对账
- 通知发布与编辑

### 4.2 用户 / 会员

- 注册、登录、个人资料修改
- 场地预约：下单、支付、取消、详情查看
- 课程预约：下单、支付、状态查看
- 器材租借：下单、支付、详情查看
- 赛事报名：报名、支付、搭档相关信息
- 穿线服务：下单、支付、取消
- 充值、充值记录、消费记录
- 搜索、通知、设置、帮助与反馈

### 4.3 教练

- 教练端登录与路由分流
- 我的课程
- 我的课表
- 预约明细与详情
- 预约状态处理
- 我的档案

### 4.4 多端一致性

对以下业务建议至少做一次 Web 与 UniApp 的交叉回归：

- 场地预约
- 课程预约
- 赛事报名
- 器材租借
- 充值与余额变化
- 通知展示

## 5. 当前建议的自动化补强顺序

### 第一批：后端回归测试

优先级最高的模块：

1. 场地预约
2. 课程预约
3. 赛事报名
4. 器材租借
5. 穿线支付
6. 充值与财务联动

### 第二批：前端关键流程冒烟

建议逐步补：

- Web 登录态与路由守卫冒烟
- 用户端关键下单页接口联调冒烟
- 教练端关键查询与状态操作冒烟

### 第三批：小程序端角色化回归模板

建议沉淀成固定检查表：

- 用户端
- 移动管理端
- 教练端

这样后续每次发版前，都能快速复用。

## 6. 当前阶段结论

2026-05-17 这次同步之后，可以比较明确地说：

- 主干的“编译 / 构建 / 类型检查 / 基础前端单测”已经是绿色
- 真正的短板已经转移到“后端自动化测试不足”和“多角色真实业务回归仍需继续补”

所以这份测试文档后续最重要的任务，不是再记更多命令，而是逐步把这些缺口变成真正的回归护栏。
