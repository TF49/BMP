## 三、模块关联与建议测试顺序

- **基础数据**：场馆 → 场地、器材、教练、赛事（均依赖场馆）。
- **会员与资金**：会员 → 充值；会员参与预约/租借/课程/赛事产生消费与财务流水。
- **业务**：场地预约（场馆+场地+会员）、课程预约（课程+会员）、赛事报名（赛事+会员）、器材租借（器材+会员）。
- **汇总**：财务管理、系统首页统计。

**建议测试顺序**（按依赖从前到后，减少“上游未准备”导致的遗漏）：

1. **第一阶段**：认证 → 用户管理 → 场馆管理 → 会员管理
2. **第二阶段**：场地管理 → 场地预约 → 器材管理 → 器材租借 → 教练管理 → 课程管理 → 课程预约 → 赛事管理 → 赛事报名
3. **第三阶段**：充值中心 → 财务管理 → 系统首页（含统计分析）
4. **独立**：穿线服务（可与器材管理一起测）
5. **教练端**：需先有 COACH 用户并绑定教练档案；登录后测工作台、我的课程、课表、预约明细、我的档案

### 模块 20：错误页（403.vue / 404.vue）

**页面入口**：无权限时跳转 `/403`，访问不存在路径时跳转 `/404`

**模块关联**：

- **与认证/用户管理**：无权限访问某页（如普通用户访问用户管理、场馆管理员访问其他场馆数据）时由路由或接口返回 403，跳转至 403 页；未登录访问需认证页时跳转登录。
- **与路由**：访问不存在的路径时跳转 404 页；“返回首页”根据当前用户角色回到管理端或用户端首页。
