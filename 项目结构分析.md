# BMP 项目结构分析

> 分析日期：2026-03-14

## 一、当前结构总览

```
BMP/
├── pom.xml                 # 后端 Maven 配置（Spring Boot 3.2 / Java 17）
├── BMP.iml                 # IntelliJ 根模块（指向 src，Maven 后端）
├── src/                     # 后端 Java 源码
│   └── main/java/com/badminton/bmp/
│       ├── modules/         # 业务模块（booking/coach/court/member/venue/...）
│       ├── config/          # 安全、缓存、Swagger、WebSocket 等
│       ├── common/          # JWT、Result、异常、校验等
│       └── websocket/       # STOMP 推送
├── vue/                     # Web 管理端（Vue 3 + Element Plus）
├── BMP-uniapp/              # 移动端 UniApp（微信小程序 / H5 / App）
├── BMPandroid/              # 移动端 Android 原生应用
├── .idea/                   # IDEA 工程配置（当前为根 Maven 模块）
├── target/                  # Maven 构建输出（可忽略）
├── logs/                    # 运行时日志（可忽略）
├── uploads/                 # 上传文件（可忽略）
├── README.md
├── 项目开发文档.md、API文档.md、数据库设计文档.md 等  # 文档散落在根目录
└── .gitignore
```

## 二、各端职责与构建方式

| 端 | 目录 | 构建/运行 | 说明 |
|----|------|-----------|------|
| 后端 | 根目录 + `src/` | Maven (`pom.xml`) | Spring Boot 3.2，端口 9090 |
| Web 管理端 | `vue/` | npm (Vue CLI) | 端口 8080，代理 /api、/ws |
| 移动端 UniApp | `BMP-uniapp/` | npm (Vite + UniApp) | 小程序 / H5 / App |
| 移动端 Android | `BMPandroid/` | Gradle (独立工程) | 用 Android Studio 打开该目录 |

## 三、与 CMS 的对比

- **CMS**：根目录同时存在 `pom.xml`、`build.gradle.kts`、`settings.gradle.kts` 等，Android 子项目在 `cmsandroid/`，结构略混合。
- **BMP**：后端仅 Maven，Android 完全收口在 `BMPandroid/`，无根目录 Gradle 文件，**职责更清晰**。

## 四、优点（无需调整）

1. **前后端分离明确**：后端 `src/` + Maven，前端多端各自独立目录。
2. **多端命名统一**：`vue`（Web）、`BMP-uniapp`、`BMPandroid`，与业务名一致。
3. **Android 独立**：`BMPandroid/` 内含完整 Gradle、app 模块、.idea，用 Android Studio 单独打开即可，不污染根目录。
4. **后端模块化**：`modules/` 按业务拆分（booking、coach、court 等），易于扩展。
5. **根 .idea**：仅管理 `BMP.iml`（Maven 模块），不掺杂 Android 配置，符合“根目录 = 后端 + 文档”的打开方式。

## 五、建议调整（可选）

### 1. 文档归拢（推荐）

- **现状**：`项目开发文档.md`、`API文档.md`、`数据库设计文档.md`、`模块化测试文档.md`、`Dashboard虚拟数据对接清单.md` 等均在根目录，根目录略显杂乱。
- **建议**：新建 `docs/`，将上述文档移入 `docs/`，根目录只保留 `README.md`。在 `README.md` 的「文档索引」中改为指向 `docs/xxx.md`。

### 2. README 与文档索引更新（推荐）

- **现状**：README 中「目录结构」和「项目状态」未包含 `BMPandroid`。
- **建议**：
  - 在「项目状态」中补充 Android 端状态（如：🚧 开发中 / ✅ 已接入等）。
  - 在「目录结构」中增加一行：`├── BMPandroid/   # 移动端 Android（Gradle，Android Studio 打开）`。
  - 若有新建 `docs/`，在「文档索引」中统一写为 `docs/项目开发文档.md` 等。

### 3. 其他（可选）

- **.gitignore**：已包含 `target/`、`logs/`、`local.properties`、`.idea` 等，无需改动。`BMPandroid` 内自有 `.gitignore`，无需在根目录为 Android 再写规则。
- **HELP.md**：若为 Maven 自动生成且不需要，可加入 `.gitignore` 或删除；若作为说明保留也可。
- **根目录不引入 Gradle**：保持当前“根 = Maven 后端”的方式即可，无需为与 CMS 完全一致而在根目录增加 Gradle。

## 六、结论

- **整体结构合理**，后端 / Web / UniApp / Android 边界清晰，Android 已正确收口在 `BMPandroid/`，**无结构性必须调整**。
- **建议做的优化**：文档归入 `docs/`、在 README 中补充 `BMPandroid` 的目录说明与状态。其余保持现状即可。
