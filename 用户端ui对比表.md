# 用户端 UI 对比表

## 文档目的

这份文档用于梳理当前 `BMP-uniapp` 用户端页面的 UI 现状，帮助后续统一设计语言，并为使用 Stitch 生成页面提供参考基线。

文档重点回答 4 个问题：

1. 当前用户端页面分成了哪些视觉风格
2. 哪些页面已经接近会长端设计语言
3. 哪些页面仍然保留旧版样式
4. 后续设计新页面时，应该向哪套风格靠拢

---

## 一、统一结论

当前用户端并不是一套统一的 UI，而是至少混合了 3 套风格：

1. 橙色品牌化风格
2. 旧版绿色业务风格
3. 半迁移混合风格

其中：

- 最接近会长端的，是首页、我的、通知中心、我的预约、登录注册、场馆相关页面
- 最不接近会长端的，是课程、器材、赛事、设置及其大部分子页面
- 后续新设计应统一向“橙色品牌化风格”靠拢，而不是继续沿用旧绿色体系

---

## 二、全局设计系统对比

| 对比项 | 用户端现状 | 会长端现状 | 问题 | 后续建议 |
| --- | --- | --- | --- | --- |
| 主色 | 同时存在橙色 `#ea580c/#ff6600/#a33e00` 和绿色 `#3cc51f` | 以橙色品牌色为主 | 视觉识别冲突，像两个产品 | 用户端统一切换到橙色品牌体系 |
| 背景 | 有的页面是浅灰白，有的是普通白底列表，有的有品牌化渐变 | 浅暖色渐变背景更统一 | 页面气质不稳定 | 统一浅色背景 + 白卡片 + 局部橙色渐变 |
| 页面壳 | 只有简单 `MobileLayout`，很多页面各自处理背景和安全区 | 已有 `PresidentLayout` 统一承载 | 用户端缺少统一容器层 | 建议新增 `MemberLayout` |
| 卡片风格 | 圆角、阴影、边框不统一 | 卡片圆角、层次较稳定 | 不同页面像不同阶段产物 | 统一卡片圆角、阴影、边框标准 |
| 导航栏 | 各页面自定义程度高，重复代码多 | 结构较统一 | 复用差，后续维护成本高 | 统一顶部导航组件 |
| TabBar | 用户端是玻璃拟态 + emoji 图标 | 会长端是品牌化专属 tabbar | 图标语义不专业，风格偏临时 | 用户端 tabbar 需要品牌化重构 |
| 组件抽象 | 很多页面自己写一套筛选栏、列表卡、标签 | 会长端已更接近系统化 | 没有形成用户端组件体系 | 优先沉淀用户端公共组件 |
| 排版 | 一部分页面有品牌标题和英文副标题，一部分还是传统列表文字 | 排版更稳定 | 信息密度和语气差异大 | 用户端统一排版节奏与标题层级 |

## 三、页面风格阵营对比

### 1. 橙色品牌化风格

| 页面 | 路径 | 当前风格判断 | 特征 | 是否可作为新设计参考 |
| --- | --- | --- | --- | --- |
| 首页 | `pages/index/index` -> `pages/index/components/MemberIndex.vue` | 强品牌化 | 顶部品牌栏、橙色 CTA、大卡片、大圆角、推荐内容区 | 是，优先参考 |
| 我的 | `pages/profile/index.vue` | 强品牌化 | 橙色头图、悬浮统计卡、会员横幅、资产中心感强 | 是，优先参考 |
| 通知中心 | `pages/notice/index.vue` | 强品牌化 | 标题区明确、卡片简洁、未读状态清晰、橙色强调 | 是，优先参考 |
| 我的预约 | `pages/booking/list.vue` | 强品牌化 | 顶部品牌栏、重点预约卡、深色统计块、预约卡片层次强 | 是，优先参考 |
| 登录 | `pages/login/login.vue` | 强品牌化 | 品牌感强、橙色渐变主按钮、视觉包装完整 | 是，优先参考 |
| 注册 | `pages/register/register.vue` | 品牌化延续 | 与登录页同系 | 是，可参考 |
| 找回密码 | `pages/recover/recover.vue` | 品牌化延续 | 与登录页同系 | 是，可参考 |
| 场馆列表 | `pages/venue/list.vue` | 品牌化较强 | 品牌标题、大图卡片、橙色信息强调 | 是，优先参考 |
| 场馆详情 | `pages/venue/detail.vue` | 品牌化较强 | 橙色视觉强调、信息分区明确 | 是，优先参考 |
| 场地预订 | `pages/venue/booking.vue` | 品牌化较强 | 预约流程感较强、操作按钮突出 | 是，可参考 |

#### 这一阵营的共同特征

- 主色为橙色品牌色
- 更像“产品型用户端”，而不是“后台型业务页”
- 强调卡片层次、信息分区、品牌标题
- 更适合成为 Stitch 新页面的参考底稿

### 3. 半迁移混合风格

| 页面 | 路径 | 当前风格判断 | 特征 | 问题 | 建议 |
| --- | --- | --- | --- | --- | --- |
| 赛事列表 | `pages/tournament/list.vue` | 半迁移 | 引入 `glass-card`，但状态色和按钮仍偏绿色 | 风格断裂 | 彻底切橙色 |
| 部分详情/流程页 | 若干用户子页 | 半迁移 | 局部用橙色，局部沿用绿色 | 视觉语言不完整 | 重新按新规范设计 |

#### 这一阵营的共同特征

- 已经试图向新体系靠拢
- 但没有完全替换旧颜色和旧组件模式
- 最容易成为“看起来不丑，但仍然不统一”的问题来源

## 五、建议作为 Stitch 参考底稿的页面

如果要用 Stitch 生成新的用户端页面，建议优先参考下面这些页面的气质，而不是参考旧绿色页面：

| 推荐参考页 | 推荐原因 |
| --- | --- |
| 首页 | 最完整地体现了新版用户端品牌感 |
| 我的 | 最能代表用户资产中心和个人中心的新版语言 |
| 通知中心 | 信息型页面的新版模板最清晰 |
| 场馆列表 | 服务列表型页面中较接近成熟产品页 |
| 场馆详情 | 详情页的新版方向较明确 |
| 登录页 | 品牌入口页完成度高 |
| 会长端我的页 | 可参考其简洁、统一、卡片化结构 |
| 会长端首页/管理页 | 可参考其浅暖背景、玻璃卡、橙色主色体系 |

## 七、后续统一设计时建议采用的标准

### 1. 色彩标准

| 角色 | 建议颜色 |
| --- | --- |
| 主品牌色 | `#ea580c` |
| 主按钮渐变 | `#a33e00 -> #ff6600` |
| 浅色强调背景 | `#fff7ed` / `#ffedd5` |
| 页面背景 | `#f8fafc` 或浅暖灰白 |
| 文字主色 | `#1e293b` / `#1a1c1c` |
| 次级文字 | `#64748b` / `#5f5e5e` |
| 边框 | `#e2e8f0` |
| 成功/状态色 | 可保留绿色，但只能作为状态辅助色，不能做主品牌色 |

### 2. 组件标准

| 组件 | 建议样式方向 |
| --- | --- |
| 顶部导航 | 简洁、留白充足、支持品牌标题 |
| 列表卡片 | 白底、大圆角、轻阴影、分层信息 |
| 主按钮 | 橙色渐变、圆角较大、固定底部优先 |
| 标签 | 胶囊形、浅底色、少量高亮 |
| 统计卡 | 可以使用悬浮卡、大数字、弱说明文字 |
| 筛选栏 | 胶囊按钮、浅灰底、激活态使用橙色 |
| 空状态 | 轻插画或图标 + 明确行动提示 |
| 底部 TabBar | 不再使用 emoji，改为统一 icon 体系 |

### 3. 版式标准

| 项目 | 建议 |
| --- | --- |
| 大标题 | 强调页面主题，允许使用品牌化标题区 |
| 二级信息 | 采用小号说明文字，降低噪音 |
| 内容区 | 卡片化分区，避免整页散点布局 |
| 留白 | 保持呼吸感，不做过度拥挤列表 |
| 安全区 | 顶部和底部安全区必须统一考虑 |

## 九、给后续 Stitch 设计时的简要原则

可以直接记住下面这几条：

1. 用户端统一参考首页、我的、通知中心、场馆页，不参考旧绿色页面
2. 所有新页面统一使用橙色品牌体系，不再使用绿色做主色
3. 优先做“产品感移动端页面”，不是“后台列表页”
4. 统一大圆角、轻阴影、白卡片、浅背景
5. 所有列表页、详情页、设置页都应纳入同一套用户端视觉系统

---

## 十、一句话结论

后续用户端 UI 设计，应以“首页 + 我的 + 通知 + 场馆页 + 会长端品牌体系”为统一参考源，彻底替换课程、器材、赛事、设置等页面中残留的旧绿色业务风格。
