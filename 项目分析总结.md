# 羽擎（BMP）项目分析总结

> **分析日期**：2026-07-16
> **项目版本**：1.0.0  
> **分析口径**：以当前仓库代码、构建结果、路由与页面注册、根目录文档为准。

## 一、阶段判断

**BMP 主业务功能完整，处于可演示、可课程/毕设交付状态；生产发布仍有明确的工程收尾项。**

依据：

- 后端 **16** 个业务模块、**30** 个模块 API Controller、**243** 个方法级 HTTP 映射、**212** 个 Java 源文件
- 数据库 **27** 张物理表，全量脚本 `badminton.sql` 可一键初始化
- Web 同时交付官网、管理端、用户端、教练端（**51** 个页面组件）
- UniApp 覆盖用户端、移动管理端、教练端（`pages.json` 注册 **107** 个页面）
- 已建立后端 **13 类 / 54 用例**和 UniApp **7 文件 / 35 用例**的自动化回归基线
- 2026-07-16 验证：后端测试、Web 生产构建、UniApp 类型检查与单测均通过

项目已从「搭骨架 / 补模块」进入「完整业务闭环 + 文档与仓库整理」阶段，适合提交 GitHub 作为作品集或课程/毕设交付物。

## 二、已交付能力清单

### 经营与资源

- 场馆 CRUD、图片、营业时间、状态日志
- 场地 CRUD、三套价格、营销开关、占用与时间轴
- 教练档案、账号绑定、场馆数据范围
- 课程、器材、赛事基础资源管理

### 交易与服务流程

- 场地预约：拼场 / 包场、主单 + 明细、支付与退款
- 课程预约、教练端预约处理
- 教练本人学员列表/详情、训练进度、课表、出勤、课程消费记录
- 课程考勤 `CHECK_IN / COMPLETE / ABSENT` 状态机与实时刷新事件
- 器材租借、赛事报名（含搭档）、穿线工单
- 会员充值、消费记录、财务流水

### 治理与运营

- 财务审计日志、财务对账
- Dashboard 汇总、近期动态聚合（无独立活动表）
- 综合搜索、通知、联系留言与反馈
- 天气代理、地图逆地理编码
- 支付超时自动取消（可配置、前后端倒计时一致）
- WebSocket 实时推送能力
- 官网匿名运营概览（场地利用率、预约成功率、器材可用数、当日收入）

### 多端与权限

- 五角色 RBAC + JWT 双 Token
- Web 四场景 + UniApp 三场景统一后端 API

## 四、已知限制（不影响功能完成度声明）

以下为**可选后续优化**，不阻碍当前版本作为完整项目发布：

| 项 | 说明 |
|----|------|
| 后端测试范围 | 54 个测试集中于近期高风险链路，预约、财务、会员等存量模块仍需角色化联调回归 |
| Web 构建告警 | 存在 CSS 顺序冲突、入口约 2.28 MiB、Browserslist 过期与 Sass `legacy-js-api` 告警 |
| 小程序 WebSocket | `/ws-native` 已注册且客户端已接入，但 `SecurityConfig` 匿名握手白名单目前只覆盖 `/ws`；上线前必须补齐并真机验证 |
| Redis 生产依赖 | `prod` profile 要求提供 `REDIS_HOST`，不能按开发环境的 Caffeine 回退方式部署 |
| 第三方 Key | 天气、地图需自行配置 API Key 后才有完整能力 |
| 演示数据 | SQL 含演示账号与业务数据，公开仓库建议脱敏 |

BMP 已实现羽毛球场馆经营场景下的**完整多端业务闭环**，并为教练学员、考勤与实时通知补上了针对性自动化测试。当前适合作为毕业设计 / 作品集交付；若面向生产，优先完成 `/ws-native` 安全放行与真机验证、扩大存量业务测试覆盖、收敛 Web 构建告警，并完成生产密钥与 Redis 配置。
