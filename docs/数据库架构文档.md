## 1. 数据库概览

### 1.1 数据库分类

BMP 三智能体系统涉及三类数据库：

1. **PostgreSQL (Agent 专用)**：存储 Agent 会话状态、Checkpoint、知识库向量
2. **MySQL (现有业务)**：存储 BMP 现有业务数据（不直接访问）
3. **Redis (缓存与限流)**：存储缓存数据、限流计数器、幂等键

### 1.2 数据库架构原则

- Agent 不直接连接 BMP 的 MySQL
- Agent 通过 Java Tool API 访问业务数据
- PostgreSQL 使用 pgvector 扩展支持向量检索
- Redis 用于高性能缓存和分布式限流

### 2.2 表结构设计

#### 2.2.1 会话表 (agent_conversations)

存储 Agent 会话基本信息。

```sql
CREATE TABLE agent_conversations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    conversation_id VARCHAR(64) UNIQUE NOT NULL,
    user_id VARCHAR(64) NOT NULL,
    agent_type VARCHAR(32) NOT NULL,
    thread_id VARCHAR(64) NOT NULL,
    venue_id VARCHAR(64),
    status VARCHAR(32) NOT NULL DEFAULT 'active',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
    metadata JSONB DEFAULT '{}',
    
    CONSTRAINT chk_agent_type CHECK (agent_type IN ('booking', 'analytics', 'support')),
    CONSTRAINT chk_conversation_status CHECK (status IN ('active', 'archived', 'deleted'))
);

-- 索引
CREATE INDEX idx_agent_conversations_user_id ON agent_conversations(user_id);
CREATE INDEX idx_agent_conversations_agent_type ON agent_conversations(agent_type);
CREATE INDEX idx_agent_conversations_thread_id ON agent_conversations(thread_id);
CREATE INDEX idx_agent_conversations_expires_at ON agent_conversations(expires_at);
CREATE INDEX idx_agent_conversations_status ON agent_conversations(status);
```

`user_id` 是现有 MySQL 用户的外部标识，只建立普通索引。PostgreSQL 不复制 MySQL
`users` 表，也不建立跨数据库外键；会话归属和用户有效性由 Java Agent 网关与 Tool 层校验。

        REFERENCES agent_conversations(conversation_id) ON DELETE CASCADE,

    CONSTRAINT chk_message_type CHECK (message_type IN ('text', 'card', 'confirmation', 'error'))

-- 索引

CREATE INDEX idx_agent_messages_created_at ON agent_messages(created_at DESC);

### 2.3 LangGraph Checkpoint 表

LangGraph 使用自己的 Checkpoint 机制，需要创建相应的表结构。

```sql
-- LangGraph Checkpoint 主表
CREATE TABLE checkpoints (
    thread_id VARCHAR(64) NOT NULL,
    checkpoint_ns VARCHAR(64) NOT NULL DEFAULT '',
    checkpoint_id VARCHAR(64) NOT NULL,
    parent_checkpoint_id VARCHAR(64),
    type VARCHAR(32) NOT NULL,
    checkpoint JSONB NOT NULL,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    PRIMARY KEY (thread_id, checkpoint_ns, checkpoint_id)
);

-- 索引
CREATE INDEX idx_checkpoints_thread_id ON checkpoints(thread_id);
CREATE INDEX idx_checkpoints_parent_checkpoint_id ON checkpoints(parent_checkpoint_id);
CREATE INDEX idx_checkpoints_created_at ON checkpoints(created_at DESC);

-- LangGraph Checkpoint 写入表
CREATE TABLE checkpoint_writes (
    thread_id VARCHAR(64) NOT NULL,
    checkpoint_ns VARCHAR(64) NOT NULL DEFAULT '',
    checkpoint_id VARCHAR(64) NOT NULL,
    task_id VARCHAR(64),
    idx INTEGER NOT NULL,
    channel VARCHAR(64) NOT NULL,
    value JSONB NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    PRIMARY KEY (thread_id, checkpoint_ns, checkpoint_id, task_id, idx)
);

-- 索引
CREATE INDEX idx_checkpoint_writes_thread_id ON checkpoint_writes(thread_id);
CREATE INDEX idx_checkpoint_writes_checkpoint_id ON checkpoint_writes(checkpoint_id);
```

## 3. Redis 数据结构设计

### 3.1 缓存数据结构

#### 3.1.1 会话缓存

```
Key: agent:session:{conversation_id}
Type: Hash
Fields:
  - user_id: 用户ID
  - agent_type: Agent类型
  - thread_id: LangGraph线程ID
  - venue_id: 场馆ID
  - status: 状态
  - expires_at: 过期时间
TTL: 3600秒 (1小时)
```

#### 3.1.2 幂等键缓存

```
Key: agent:idempotency:{idempotency_key}
Type: String
Value: JSON格式的执行结果
TTL: 86400秒 (24小时)
```

#### 3.1.3 限流计数器

```
Key: agent:ratelimit:{user_id}:{type}
Type: String
Value: 计数器值
TTL: 60秒 (1分钟)

Key: agent:ratelimit:{ip}:{type}
Type: String
Value: 计数器值
TTL: 60秒 (1分钟)
```

#### 3.1.4 工具结果缓存

```
Key: agent:tool_cache:{tool_NAME}:{params_hash}
Type: String
Value: JSON格式的工具结果
TTL: 300秒 (5分钟)
```

#### 3.1.5 用户预算缓存

```
Key: agent:budget:{user_id}:{date}
Type: Hash
Fields:
  - tokens_used: 已使用Token数
  - requests_count: 请求次数
TTL: 86400秒 (24小时)
```

### 3.2 分布式锁

```
Key: agent:lock:{resource_id}
Type: String
Value: 锁持有者标识
TTL: 30秒
```

## 5. 数据迁移策略

### 5.1 版本管理

使用 Alembic 进行 PostgreSQL 数据库版本管理。

```python
# alembic/versions/001_initial_schema.py
from alembic import op
import sqlalchemy as sa
from sqlalchemy.dialects import postgresql

def upgrade():
    # 创建扩展
    op.execute('CREATE EXTENSION IF NOT EXISTS vector')
    
    # 创建表
    op.create_table(
        'agent_conversations',
        sa.Column('id', postgresql.UUID(as_uuid=True), primary_key=True),
        sa.Column('conversation_id', sa.String(64), unique=True, nullable=False),
        sa.Column('user_id', sa.String(64), nullable=False),
        sa.Column('agent_type', sa.String(32), nullable=False),
        sa.Column('thread_id', sa.String(64), nullable=False),
        sa.Column('venue_id', sa.String(64)),
        sa.Column('status', sa.String(32), nullable=False, server_default='active'),
        sa.Column('created_at', sa.TIMESTAMP(timezone=True), server_default=sa.text('CURRENT_TIMESTAMP')),
        sa.Column('updated_at', sa.TIMESTAMP(timezone=True), server_default=sa.text('CURRENT_TIMESTAMP')),
        sa.Column('expires_at', sa.TIMESTAMP(timezone=True), nullable=False),
        sa.Column('metadata', postgresql.JSONB(), server_default='{}'),
    )
    
    # 创建索引
    op.create_index('idx_agent_conversations_user_id', 'agent_conversations', ['user_id'])
    op.create_index('idx_agent_conversations_agent_type', 'agent_conversations', ['agent_type'])
    op.create_index('idx_agent_conversations_thread_id', 'agent_conversations', ['thread_id'])
    op.create_index('idx_agent_conversations_expires_at', 'agent_conversations', ['expires_at'])
    op.create_index('idx_agent_conversations_status', 'agent_conversations', ['status'])

def downgrade():
    op.drop_table('agent_conversations')
    op.execute('DROP EXTENSION IF EXISTS vector')
```

#### 向量相似度搜索

```sql
SELECT 
    kd.chunk_id,
    kd.content,
    kd.document_id,
    1 - (kd.embedding <=> '[0.1,0.2,0.3,...]') as similarity
FROM knowledge_chunks kd
JOIN knowledge_documents doc ON kd.document_id = doc.document_id
WHERE doc.status = 'active'
ORDER BY kd.embedding <=> '[0.1,0.2,0.3,...]'
LIMIT 10;
```

#### 监控查询

```sql
-- 表大小监控
SELECT 
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- 慢查询监控
SELECT 
    query,
    calls,
    total_time,
    mean_time,
    max_time
FROM pg_stat_statements
ORDER BY mean_time DESC
LIMIT 10;

-- 索引使用监控
SELECT 
    schemaname,
    tablename,
    indexname,
    idx_scan,
    idx_tup_read,
    idx_tup_fetch
FROM pg_stat_user_indexes
ORDER BY idx_scan DESC;
```
